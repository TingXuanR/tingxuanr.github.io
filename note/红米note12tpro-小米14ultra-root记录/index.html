红米Note12TPro 小米14Ultra Root记录

发表于： 2025-02-05 更新于： 2025-07-17 分类于： 笔记/玩机记录
字数： 1979 阅读：≈ 4分钟 浏览： 评论：
摘要

记录一下给红米note12t pro、红米note11t pro、小米14ultra的root过程。推荐用狐狸面具或KSU，隐藏root能过电信app和银行app的检测。联发科芯片和骁龙芯片及不同安卓版本的root过程有差异。

红米Note 12T Pro Root记录

Root

准备工作：

  • 手机解BL锁(目前小米系手机miui系统可解，hyper系统2025年之后花钱可秒解；一加截至20250205可以申请解锁；解BL锁时会清除手机数据。)

  • 手机开启usb调试，usb调试（安全）设置

  • 手机连接电脑

  • 电脑准备好adb-fastboot, payload-dumper-go-64

  • 手机下载好狐狸面具( https://magiskdelta.com/apk/ )

  • 退出手机账号! (尤其是谷歌账号)

  1. 在手机设置里检查系统更新，选择下载最新完整系统包更新。

  2. 将手机Dowload->downloaded_rom目录下找到下载的更新包，复制到document目录。防止系统更新完后更新包被删除。

  3. 将更新包复制到电脑，解压。

  4. 将解压出来的payload.bin复制到payload目录下。

  5. 运行bat脚本，选择解压所有img。其实用到的就两个，一个boot.img, 一个vbmeta.img。

  6. 将boot.img复制到手机Download目录，狐狸面具首页点击“安装”，选择“选择并修补一个文件”。就会得到一个magisk_patched-***的文件，把这个文件复制到电脑上adb-fastboot所在的目录。

  7. 点击bat脚本，打开cmd窗口。手机关机，重启到fastboot模式。不要用cmd命令fastboot reboot重启，否则就会到fastbootd模式。关机后按音量减键+电源键进入fastboot模式。

  8. 依次输入以下命令：

    # 1. 检查设备是否正常连接
fastboot devices 
# 2.1 刷入修补后的boot文件（redmi note11t/12t）
fastboot flash boot magisk_patched-27001_PTnl0.img
# 2.2（二选一） 如果修补的是init_boot（小米14ultra）
fastboot flash init_boot magisk_patched-27001_PTnl0.img
# 3.（可选） 如果用的是联发科天玑8100芯片。
fastboot --disable-verification flash vbmeta vbmeta.img
# 重启
fastboot reboot

  9. 顺利的话，root就完成了。如果卡mi了，或则卡miui的recovery。那么可以重新fastboot flash boot boot.img，把原来的boot.img刷入重启。或者清除全部数据后重启。

  10. 有些时候可能还需要关闭vbmeta验证( https://www.coolapk.com/feed/34454207?shareKey=ZTQxNjFiNzc4YjJhNjI2YTViZDM~&shareUid=1361589&shareFrom=com.coolapk.market_12.2 )，此时就需要在reboot前多输入一条fastboot –disable-verification flash vbmeta vbmeta.img

root完后最好在magisk delta里开启“Zygisk”, “Magisk Hide”, “实现SuList”，这样可以比较好的隐藏root。只有在SuchList名单里的app才能获取root权限。

小米14Ultra Root记录

与12TPro相似，区别在于骁龙芯片不需要关闭vbmeta验证；hyperOS用的安卓14，需要修补的是init_boot.img。

准备工作、1-4步一致。

  1. 选择解压init_boot.img即可

  2. init_boot.img复制到手机Download目录，狐狸面具首页点击“安装”，选择“选择并修补一个文件”。就会得到一个magisk_patched-***的文件，把这个文件复制到电脑上adb-fastboot所在的目录。

  3. 手机关机后按音量减键+电源键进入fastboot模式重启到fastboot模式。**不要用cmd命令fastboot reboot重启，否则就会到fastbootd模式。**点击bat脚本，打开cmd窗口。

  4. 依次输入以下命令

    1
2
3
4
5
6

    # 检查设备是否正常连接
fastboot devices 
#如果修补的是init_boot则是
fastboot flash init_boot magisk_patched-26400_wOw4V.img
# 重启
fastboot reboot

  5. 如果adb devices没有反应，没关系。fastboot devices能出现设备信息也能正常root。

使用delta面具（改名成了26.4-kitsune-2：https://github.com/HuskyDG/magisk-files）后，中国电信和银行这种之前打不开的软件都能打开了。

从澎湃1.0升级到2.0掉Root的问题

看酷安很多人都能够保root升级2.0，但是全量更新后却还是掉Root了，更严重的是修补root后，magisk beta还是显示无root。唯一能够怀疑就是在全量更新重启前，关掉了所有模块，可能导致一些冲突问题。

还好试了下，用Kernel Su修补后能够正常root。需要注意是Kernel Su虽然兼容magisk模块，但是并没有原生支持zygisk。需要安装https://github.com/Dr-TSNG/ZygiskNext 之后才能使存储空间增强模式依赖的模块生效。
